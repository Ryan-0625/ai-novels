# AI-Novels — Multi-Agent 智能小说生成系统

---

## 1. 项目介绍

**AI-Novels** 是一个基于 **Multi-Agent 架构** 的 AI 原生小说生成平台，实现了从故事构思到完整章节生成的一站式创作流程。系统通过协调 **10+ 专业 AI Agent**（角色设计、世界观构建、大纲规划、内容生成、质量检查、人性化润色等）协同工作，采用 **DAG 有向无环图** 任务编排引擎进行智能调度，支持异步并行章节生成与依赖管理。

后端基于 **Python FastAPI** 构建，前端采用 **Vue 3 + TypeScript + Element Plus** 实现玻璃拟态响应式界面。系统集成 **DeepSeek、OpenAI、Ollama、通义千问、Gemini、MiniMax** 等多种 LLM，支持运行时切换；使用 **PostgreSQL + MongoDB + Neo4j + ChromaDB + Qdrant** 异构存储体系；通过 **Event Bus + Redis Streams** 实现组件解耦与实时事件推送，SSE 实时推送 Agent 执行日志到前端。项目已容器化部署（Docker Compose），配置 **Prometheus + Grafana** 监控体系，并支持 Kubernetes 编排。

## 3. 项目亮点

### ① Multi-Agent 智能协作架构

设计了 10+ 个专业化 AI Agent（ConfigEnhancer、OutlinePlanner、CharacterGenerator、WorldBuilder、ContentGenerator、QualityChecker、Humanizer 等），每个 Agent 独立负责创作流水线中的一个环节，通过 Coordinator Agent 统一编排调度，实现端到端的自动化小说生成。

### ② DAG 有向无环图任务编排引擎

自研基于 DAG 的工作流引擎，支持并行节点执行、依赖关系管理、非关键节点失败自动跳过与重试机制。ContentGenerator 利用并发特性同时对多个章节进行生成，大幅提升长篇小说创作效率。

### ③ 事件驱动架构与实时可视化

基于 Event Bus + Redis Streams 实现组件间解耦通信，所有 Agent 执行状态（started / completed / failed / skipped）及 LLM 调用日志通过 SSE 实时推送至前端，用户可在 TaskMonitor 中以时间线形式查看完整的创作进度。

### ④ 异构多模态数据存储策略

根据业务场景选择最优存储：Neo4j 存储角色关系图谱、MongoDB 存储章节正文、PostgreSQL 存储任务元数据、ChromaDB / Qdrant 双向量库支撑语义检索，Redis 承担缓存与事件流，充分利用各类数据库的领域优势。

### ⑤ 多 LLM 提供商标配与运行时切换

对接 6 种主流 LLM 提供商（DeepSeek、OpenAI、Ollama、通义千问、Gemini、MiniMax），通过统一的 LLMRouter 抽象层实现运行时热切换。Agent 可在不同阶段选择不同模型（如规划阶段用高性价比模型、生成阶段用强推理模型），兼顾成本与质量。

### ⑥ 全链路 DevOps 与监控体系

完整容器化部署支持（Docker Compose multi-profile），配置 Prometheus + Grafana 实时监控系统性能，GitHub Actions CI/CD 自动化测试与镜像构建，Kubernetes manifests 已就绪。项目通过 260+ 测试用例覆盖核心模块，保障系统稳定性。
